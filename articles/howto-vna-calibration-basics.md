## 1. RFのための測定機たち

高周波の電気信号を測る装置には、様々なものがあります。
特に有名な装置は以下の三種類ですね。

| 測定機 | 主な用途 |
| --- | --- |
| オシロスコープ | 時間波形 |
| スペクトラムアナライザ | 周波数スペクトル |
| VNA | 回路の伝送特性 |

この中でも、VNAは回路の入出力を測定する装置です。

たとえば、こんなときに使います。
- フィルタが通す周波数はどこからどこまで？
- アンプの利得はいくつ？
- コネクタと基板はマッチングしてる(反射はない)？

自ら信号を出さない回路（パッシブ回路）が、ある範囲の周波数帯でどんな特性を持っているかを測る道具です。

## 2. 伝送特性

VNAで回路を評価するとき、知りたいのはたいてい次のような特性です。

-  挿入損失 (= Insertion Loss)
    - 回路を通過したときに、失われる電力の程度
-  反射 (= Return Loss)
    - 回路に入れた信号の、戻ってくる電力の程度
-  クロストーク(⇔ Isolation)
    - ある回路に信号を入れたとき、隣の回路に漏れてしまう電力の程度

これらを測定したとき、VNAの画面では縦軸がdB、横軸がHzで表現されることが多いですが、
一方でスミスチャートでマッチング取るときとかは、ぜんぜん違う表現ですね。
でも、どちらもファイルとして出力すると、Sパラメータなるもので出てきます。

## 3. なぜSパラメータ？

Sパラメータ自体の詳しい解説は、検索すれば山のように出てくるので、ここでは省きます。
ただ、高周波だとなぜSパラを使うのかという話だけ。

- 測定しやすい（測定機目線）
    - 高周波の小信号では、電圧と電流の正確な測定が難しい。
    - インピーダンスが決まっていれば電力は測れる。
- 扱いやすい（計算機目線）
    - 損失や反射、スミスチャートなど好きな形に変換できる。
    - データ容量も軽い。
- 応用が効く（ユーザー目線）
    - 回路シミュレータにもそのまま放り込める。
    - 電磁界シミュレータの出力にも使われていて、実測と比較しやすい。

## 4. VNAはどうやって測っているのか

我々がDUT（測定対象物）をVNAに接続し、VNAに「Portは1と2を使い、周波数は1から10 GHzまで1 GHzステップで測定」と指示したとき、VNAは次のことを行っています。

1.  Port1から1 GHzの信号を出す
2.  DUTで反射してPort1に返ってきた電力を測る (S11)
3.  DUTを通過してPort2に到達した電力を測る (S21)
これをPort2から出す逆バージョンも含め、2,3,...,10 GHzまで自動でやってくれます。

![S-parameter image](/images/vna-calibration/article-01/s-parameter-measurement.png)

## 5. 実際の測定系

ただし、実際の測定では目指すDUT特性だけでなく、測定系全体の特性が含まれます。
それらの「誤差」は山のようにあって、以下はその一部です。

1. 反射
   - VNA内部回路
   - port–ケーブル接続
   - ケーブル内部
   - ケーブル–DUT接続

2. 漏れ
   - VNA内部回路
   - port間

3. 減衰
   - VNA内部回路
   - ケーブル

4. 位相ずれ
   - VNA内部回路
   - ケーブル

![Error image](/images/vna-calibration/article-01/measurement-error-sources.png)

これら誤差すべてが、測定結果に含まれます。
人間がエクセルなんかで計算して取り除くのは、ちょっと現実的ではないですね。

## 7. エラーボックスモデル

そんな複雑きわまる誤差をシンプルに扱うための、Error Boxという数学モデルがあります。
VNAの内部回路、Port、ケーブル、コネクタなんかの誤差を全てひっくるめて箱に入れてしまおうというものです。

![Error Box image](/images/vna-calibration/article-01/error-box-model.png)

このモデルにより、VNAはどんな測定系による誤差でも、いつも同じような計算で処理できるようになります。
私たちが校正画面をぽちぽちクリックしている裏で、VNA内部ではError Boxが作成されていきます。
そして、出来上がったError Boxを実測値から外すことで、VNAはきれいなデータを表示してくれます。

## 8. 校正とは何か

まとめに入ります。
校正とはError Boxという箱に測定系の誤差をまとめて放り込み、それを数学的に消す作業です。

測定した生データ
Error Box1 (Port1側の誤差) + DUT + Error Box2 (Port2側の誤差)
↓
校正で得たError Boxを引く
↓
DUT

もちろん、VNA内部では複雑な計算をしているわけですが、私たちユーザーはこれくらいの粒度で認識しておけばよいと思います。
